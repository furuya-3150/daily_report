## わかったこと
- docker run —volume-from <コンテナID>でコンテナ間でvolumeすることができるが、エフェメラルになるように設計することが好ましいので、基本的にVolumeは使用しない。
### dockerデバック
- コンテナが何かしらの理由で死んでしまうことがあり、コンテナに入りデバックすることができない場合、`docker commit <container id> <コンテナ名>`で実行中のコンテナからイメージを作成できるので、そのイメージからコンテナを起動してバグの調査を行う。
### docker ベストプラクティス
- VMのように1イメージに対してCentOS,Mysqll, Nginxをインストールするのではなく、1コンテナに対して1プロセスとしエフェメラルな設計を意識する。

### マルチステージビルド
- Dockerfileはできるだけ軽量化されていることがベストプラクティスになり、プロジェクト立ち上げ時以降はビルド用にインストールされたパッケージは不必要になる。Dockerfileにビルド時に必要なイメージ、実行時に必要なイメージをステージ別に記載することで、イメージを軽量化することができる。
