## わかったこと
### socket
- プログラムとネットワークを繋げるなにか。docker CLIを使用してソケット経由でdockerデーモンにアクセスする。新しいサーバーを構築し、dockerコマンドを実行するときはdockerグループにログインしているユーザーを追加する必要がある`sudo gpasswd -a $USER docker`そうするとdocker.sockにアクセスできるのでdockerコマンドをsudoコマンドなしに実行することができる。

### ssh
- ネットワーク経由で安全にリモートマシンに接続するプロトコル。リモートマシンに接続して、マシンの操作を行うことができる。

### SFTP(SSH File Transfer Protocol or Secure File Transfer Protocol)
- インターネット上でファイルの転送を安全に行うためのプロトコル。リモートサーバーとの安全なやり取りとしては同じであるが、ファイル転送に特化したものがSTFPとなる。

### CPUのアーキテクチャ
- arm:appleシリコンの場合armアーキテクチャのCUP
- adm:intelの場合amdアーキテクチャのCUP。x86だと32bitのadmアーキテクチャになるので、4GBのメモリを確保できる。x64はだと64biのadmアーキテクチャになるので、16GBのメモリを確保できる。

### デ ィスク
- マシンにはディスク容量があらかじめ設けられているが、パーテーションがありそれぞれの用途によって容量が分けられている。ディスクの容量を増やしたら`sudo growpart /dev/xvda 1`で特定のディスクパーテーションの特定の番号（今回は1）に対して要領を割りてる。
