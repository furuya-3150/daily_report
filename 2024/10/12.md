## わかったこと
- useState
useStateにおけるset*関数はコンポーネントが読み込まれた際に実行されるではなくバッチ化されて読み込まれるので下記のように同じ値に対して変更を行っても最後の変更しか適応されない
```
() => {
    setNum(num + 1);
    setNum(num + 1);
  }
```

具体的には<br>
最初のset*関数が読み込まれるバッチに予約される
次のset*関数が読み込まれバッチに予約される
上記のどちらも予約されているだけで実行じに numの値が変更されるわけではなく、numの値がコピーされてバッチで実行されるため2で予約された処理の num は+1される前の値であるので結果的に numには+2ではなく+1された値が格納される    

本当に現在の値を使いたいときはset*関数にコールバック関数を指定する
```
() => {
      setNum((prev) => { prev + 1 }
      setNumb((prev) => { prev + 1}
}
```

- 親コンポーネントが再レンダリングがこコンポーネントも再レンダリングされる
- useEffect: 
再レンダリング時、第2引数の配列の要素が変更されたとき第一引数(CallBck Fn)が実行されるようになる
